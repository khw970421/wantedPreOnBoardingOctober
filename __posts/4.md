트리쉐이킹 : 사용되지 않는 코드를 제거하기 위해 JavaScript 컨텍스트에서 일반적으로 사용되는 용어 (필요한 것만 가져다 쓰기)
* 예를 들어 foo 라는 패키지가 있고 해당 패키지의 foo.search 라는 메서드가 필요하다면 foo 를 import 하는게 아니라 구조분해할당을 이용해 search 메서드만을 가져오는 식으로 코드를 작성
`import foo from 'foo'` (X)
`import { search } from 'foo'` (O) 

React의 재조정 :  **가상 DOM의 각 노드가 변경되었는지 여부를 하나씩 다 순회**하면서 변화 이전의 트리와, 변화 이후의 트리가 가진 모든 속성을 하나씩 비교해볼 수 없으니, 나름의 규칙을 만들어 효율적으로 처리하는 과정 
(기존의 VDOM과 변경사항이 생긴 VDOM 의 비교작업)

코드스플리팅 : 코드 스플리팅을 하게되면, 지금 당장 필요한 코드가 아니라면 따로 분리시켜서, 나중에 필요할 때 불러와서 사용할 수 있다. 이를 통해 페이지의 로딩 속도를 개선
(모든 페이지에 대한 js는 무겁지만 하나의 페이지에 대한 js만 있으면 그 페이지에 대한 동작은 하면서 js가 가벼워서 속도 개선)

RSS : 뉴스나 블로그 사이트에서 주로 사용하는 콘텐츠 표현 방식
토스 사이트에서 https://toss.tech/ 
`<link rel="alternate" title="토스 기술 블로그, 토스테크 글 목록" type="application/rss+xml" href="/rss.xml">` 로
rss.xml을 확인할 수 있고 이를 통해 크롤링없이 `https://toss.tech/rss.xml` 를 통해 데이터를 얻어낼수 있다.
